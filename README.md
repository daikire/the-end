# the-end

Personal project workspace organized by role-based folders.

## Folder Structure

### 01_ceo
Project vision, goals, and decision records. Also holds business-related materials such as budgets, revenue plans, and contracts.

### 02_secretary
Schedule management, meeting notes, and task tracking. Also stores templates, boilerplate documents, and contact lists.

### 03_engineering
Source code, technical design docs, and architecture notes. Also includes automation scripts, tool configs, and dev environment setup.

### 04_marketing
SNS content, brand guidelines, logos, and promotional materials. Also covers ad creatives, landing pages, email campaigns, and market analysis.

### 05_pm
Project management artifacts like task boards, milestones, and progress reports. Also holds product specs, user stories, and requirements definitions.

### 06_research
Market research, competitor analysis, and trend reports. Also includes technical investigations, learning notes, and reference materials.

### 07_reviews
Project retrospectives, KPT sessions, and periodic reviews. Also collects user feedback, test results, and quality checks.

## Naming Conventions

- Use lowercase with hyphens for file names (e.g., `project-plan.md`)
- Avoid special characters: `\ / : * ? " < > |`
- Keep file paths under 200 characters for cross-platform compatibility

# Secretary フォルダ — 秘書機能

> 日々の業務ログ、気づき、会議記録、テンプレート、スケジュール管理

## このフォルダの目的

秘書のような役割で、日々の業務実行に必要な運用記録を保管します。日記、会議での議論内容、突出した気づきやアイデア、よく使うテンプレート、スケジュール計画などを記録し、日常業務の効率化と透明性確保を実現します。

## サブフォルダ構成

| フォルダ | 用途 |
|---------|------|
| diary/ | 日誌、日々の業務ログ |
| insights/ | 重要な気づき、アイデア、学習メモ |
| meetings/ | 会議記録、議論要約、アクションアイテム |
| templates/ | メール雛形、報告書テンプレート、チェックリスト |

## ファイル命名規則

- `diary_YYYYMMDD.md` — 日誌（日付ごと）
- `insight_YYYYMMDD_[テーマ].md` — 気づき・学習（日付とテーマ）
- `meeting_YYYYMMDD_[参加者/会議名].md` — 会議記録
- `template_[用途].md` — テンプレート（用途を明記）

## 運用ルール

- 日誌は営業日毎に記録する。未記録日があれば翌営業日にまとめる
- 会議後、当日中または翌営業日に記録を完成させる
- テンプレートは改善を通じて進化させ、バージョン管理を行う
- 気づきは生まれた時点で記録し、後から体系化する

## 知見の蓄積方法

日誌から月ごとに重要な出来事や課題を抽出します。複数回出現する課題パターンをinsights/に昇格させます。テンプレートは使用実績に基づき改善し、組織内で最適化されたプロセスを蓄積します。

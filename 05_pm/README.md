# PM フォルダ — プロジェクト管理（コア）

> インシデント追跡、なぜなぜ分析、設計ドキュメント改善、タスク管理

## このフォルダの目的

このフォルダは証券代行業務のPMOとしての核となる領域です。発生したインシデントの追跡、なぜなぜ分析による根本原因究明、再発防止策の立案実行、設計ドキュメント改善活動の進捗管理を行います。品質向上と業務プロセスの継続改善を推進する中心地です。

## サブフォルダ構成

| フォルダ | 用途 |
|---------|------|
| incidents/ | インシデント報告書、なぜなぜ分析、再発防止策 |
| design-improvement/ | 設計ドキュメント改善、プロセス改善提案 |
| tasks/ | 進行中のタスク、チェックリスト |
| specs/ | 仕様書、業務フロー図、要件定義 |
| archive/ | 完結したインシデント、クローズ案件 |

## ファイル命名規則

- `incident_YYYYMMDD_[概要]_v[版]_status.md` — インシデント報告書
  - status: draft/analysis/prevention/closed
- `5why_YYYYMMDD_[インシデントID].md` — なぜなぜ分析シート
- `design_improvement_[対象]_YYYYMMDD.md` — 設計改善提案
- `task_[優先度]_[内容].md` — タスク管理
- `spec_[プロセス名]_v[バージョン].md` — 仕様書

## 運用ルール

- インシデント報告は発生後24時間以内に初版を記録する
- なぜなぜ分析は最低3段階、可能な限り5段階以上を目指す
- 再発防止策は実行日、担当者、確認日を明記する
- 設計改善は月1回以上、重大インシデントが発生したら即日開始する
- すべてのインシデント完結時にarchive/へ移行し、lessons-learned/へ抽象化する

## 知見の蓄積方法

インシデントから抽出した根本原因と再発防止策を「なぜなぜ分析テンプレート」と「再発防止策チェックリスト」として体系化します。類似インシデントの再発パターンを分析し、設計段階での予防措置に反映させます。

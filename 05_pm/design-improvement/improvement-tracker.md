### カテゴリ定義

| カテゴリ | 内容 | 例 |
|---------|------|---|
| 構成・体裁 | ドキュメントの構成、フォーマット、章立て | 章構成がバラバラ、テンプレがない |
| 記載内容 | 記載すべき情報の過不足 | IF仕様の記載粒度が不統一 |
| 更新運用 | 更新ルール、版管理、レビュープロセス | 更新タイミングが不明確 |
| 命名・用語 | 用語統一、命名規則 | 同じ概念に複数の呼称がある |
| ツール・環境 | 使用ツール、保管場所 | ファイルの場所がわかりにくい |

## 優先順位づけ

### 評価軸

| 評価軸 | 重み | 説明 |
|--------|------|------|
| 影響範囲 | 高 | 多くのメンバー・ドキュメントに影響するか |
| 改善コスト | 中 | 改善にかかる工数・難易度 |
| 緊急度 | 中 | 今すぐ対応が必要か |
| 効果の大きさ | 高 | 改善後の品質向上・効率化のインパクト |

### 優先度マトリクス

| 優先度 | 条件 | 対象 # |
|--------|------|--------|
| 高（すぐやる） | 影響大 × コスト小 | |
| 中（計画的に） | 影響大 × コスト大、または影響中 × コスト小 | |
| 低（余裕があれば） | 影響小 | |

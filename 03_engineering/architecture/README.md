# 03_engineering/architecture — ITアーキテクト視点

PMO業務・ITコンサル案件に対して、技術部門の専門家（ITアーキテクト）として意見・提言を記録するフォルダ。

---

## このフォルダの役割

Daikiさんが直面する技術的課題に対して、Claudeがアーキテクト視点で分析・提言を行い記録する。

具体的には：
- システム設計の妥当性評価
- 障害の技術的根本原因の深掘り（PMO視点では捉えきれない部分）
- 外部設計書の技術品質チェック
- IT基盤・インフラの改善提案
- 証券代行業務システム特有の技術課題への対応

---

## フォルダ構成

```
architecture/
├── adr/          アーキテクチャ決定記録（Architecture Decision Records）
├── reviews/      技術レビュー・設計評価レポート
└── consulting/   ITコンサル視点の提言・分析
```

---

## 使い方

「アーキテクト視点で見て欲しい」「技術的に正しいか判断して欲しい」と言えば、
Claudeがこのフォルダに分析レポートを作成する。

---

## アーキテクト視点でできること・できないこと

**できること**
- 設計書・仕様書の技術的妥当性レビュー
- システム間の依存関係・リスク分析
- 障害の技術的メカニズムの解説
- 改善案・代替案の提示と比較
- セキュリティ・パフォーマンス・保守性の観点での評価

**できないこと**
- 実際のシステムへのアクセス・操作
- 最新の社内システム仕様の把握（共有してもらえれば分析可能）
- コードの実行・テスト（ファイルとして見ることは可能）

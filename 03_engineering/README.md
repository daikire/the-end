# Engineering フォルダ — 技術開発

> 自動化スクリプト、ツール、技術ドキュメント

## このフォルダの目的

PMO業務を効率化するための技術資産を蓄積します。分析スクリプト、報告書生成ツール、インシデント自動集計スクリプト、データ処理の自動化など、証券代行業務の効率化に直結するツールを開発・保守します。技術的な生産性向上の中枢です。

## サブフォルダ構成

| フォルダ | 用途 |
|---------|------|
| scripts/ | Python、Bash、その他スクリプト |
| docs/ | 技術仕様書、API仕様、アーキテクチャドキュメント |
| tools/ | 独立して動作するツール、実行可能ファイル |

## ファイル命名規則

- `script_[機能名].py` — スクリプト（言語拡張子を含める）
- `doc_[対象システム]_[バージョン].md` — 技術ドキュメント
- `tool_[ツール名]_v[バージョン]/` — ツール用ディレクトリ

## 運用ルール

- すべてのスクリプトには実行前に動作確認とテストを記録する
- ドキュメントはコード変更に追随する。古いドキュメントは削除しない、バージョン管理する
- 外部依存ライブラリはrequirements.txtで明記し、バージョン固定する
- セキュリティ関連のスクリプトはコードレビュー後に本番利用する

## 知見の蓄積方法

各スクリプトの実行結果、処理時間、改善前後のパフォーマンス改善率を記録します。頻繁に改善されるスクリプトから成功パターンを抽出し、他のツール開発に活用します。

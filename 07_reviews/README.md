# Reviews フォルダ — 振り返りと学習

> レトロスペクティブ、教訓、品質チェックリスト

## このフォルダの目的

インシデント対応の完結後、プロジェクト段階完了後、定期的な振り返りを通じて、組織全体の学習を加速させます。個別事象から抽出した教訓を構造化し、組織のナレッジベース化します。品質チェックリストの継続改善を通じて、予防文化を醸成します。

## サブフォルダ構成

| フォルダ | 用途 |
|---------|------|
| retrospectives/ | プロジェクト振り返り、定期レビュー記録 |
| feedback/ | ステークホルダーフィードバック、意見集約 |
| checklists/ | 品質チェックリスト、レビューリスト |
| lessons-learned/ | 組織ナレッジベース、教訓テンプレート |

## ファイル命名規則

- `retro_[プロジェクト名]_YYYYMMDD.md` — レトロスペクティブ
- `retro_monthly_YYYYMM.md` — 月次振り返り
- `feedback_[ソース]_YYYYMMDD.md` — フィードバック記録
- `checklist_[対象プロセス]_v[バージョン].md` — チェックリスト
- `lesson_[教訓カテゴリ]_[要約].md` — 教訓（lessons-learned/）

## 運用ルール

- インシデント完結から1週間以内にレトロスペクティブを実施する
- 月次振り返りは毎月末に実施し、次月の改善方針を明記する
- チェックリストは使用実績に基づき、四半期ごとに改善する
- 教訓は「背景→学習内容→適用場面→注意点」の構造で記録する

## 知見の蓄積方法

複数のレトロスペクティブから抽出した共通パターンを「教訓」として昇格させます。同じ失敗の再発を防ぐための予防チェックリストを自動生成します。組織内の教訓はwiki化し、全員がアクセス可能な状態を維持します。
